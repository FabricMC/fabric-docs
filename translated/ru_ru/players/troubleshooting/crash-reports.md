::: tip

Если у вас возникли какие-либо трудности с поиском причины сбоя, вы можете обратиться за помощью в [Fabric Discord](https://discord.fabricmc.net/) в каналах `#player-support` или `#server-admin-support`.

Отчёты о сбоях являются важной частью устранения проблем с вашей игрой или сервером. Они содержат много информации об ошибках и могут помочь вам найти причину краша.

## Поиск отчётов о сбоях {#finding-crash-reports}

Отчёты о сбоях находятся в папке `crash-reports` в папке вашей игры. Если у вас сервер, то они хранятся в папке `crash-reports` в папке сервера.

Для неофициальных лаунчеров вам следует посмотреть в их документацию, чтобы найти отчёты о сбоях.

Отчёты о сбоях можно найти по этим путям:

::: code-group

```text:no-line-numbers [Windows]
%appdata%\.minecraft\crash-reports
```

```text:no-line-numbers [macOS]
~/Library/Application Support/minecraft/crash-reports
```

```text:no-line-numbers [Linux]
~/.minecraft/crash-reports
```

:::

### Разделы отчёта о сбое {#crash-report-sections}

Отчёты о сбоях состоят из нескольких разделов, каждый из которых отделён заголовком:

- `---- Minecraft Crash Report ----` — краткое описание отчёта. Этот раздел содержит основную ошибку, вызвавшую сбой, время её возникновения и соответствующий стек вызовов. Это самый важный раздел отчёта о сбое, так как стек вызовов обычно содержит ссылки на мод, вызвавший сбой.
- `-- Last Reload --` — этот раздел не особенно полезен, если только сбой не произошёл во время перезагрузки ресурсов (<kbd>F3</kbd>+<kbd>T</kbd>). Этот раздел содержит время последней перезагрузки и соответствующий стек вызовов всех ошибок, возникших в процессе перезагрузки. Эти ошибки обычно вызваны ресурс-паками и могут быть проигнорированы, если только они не вызывают проблем с игрой.
- `-- System Details --` — этот раздел содержит информацию о вашей системе, такую как операционная система, версия Java и объём памяти, выделенный игре. Этот раздел полезен для определения того, используете ли вы правильную версию Java и достаточно ли памяти выделено игре.
  - В этом разделе Fabric добавляет специальную строку `Fabric Mods:`, за которой следует список всех установленных модов. Этот раздел полезен для определения возможных конфликтов между модами.

### Разбор отчёта о сбое {#breaking-down-the-crash-report}

Теперь, когда мы знаем, что представляет собой каждый раздел отчёта о сбое, мы можем начать разбирать его и находить причину сбоя.

## Сбои Mixin {#mixin-crashes}

::: info

Mixin'ы — это способ для модов изменять игру без необходимости изменять исходный код самой игры. Они используются многими модами и являются очень мощным инструментом для разработчиков модов.

:::

Когда mixin вызывает сбой, в стеке вызовов обычно указывается сам mixin и класс, который он изменяет.

Method mixin'ы будут содержать `mod-id$handlerName` в стеке вызовов, где `mod-id` — это идентификатор мода, а `handlerName` — имя обработчика mixin.

```text:no-line-numbers
... net.minecraft.class_2248.method_3821$$$mod-id$handlerName() ... // [!code focus]
```

Вы можете использовать эту информацию, чтобы определить мод, вызвавший сбой, и сообщить о нём автору мода.

## Что делать с отчётами о сбоях? {#what-to-do-with-crash-reports}

Лучше всего загрузить отчёт о сбое на сайт для вставки текста (paste site), а затем поделиться ссылкой с автором мода, например, через их систему отслеживания ошибок или с помощью какого-либо канала связи (Discord и др.).

Это позволит автору мода исследовать сбой, при необходимости воспроизвести его и устранить причину проблемы.

Часто используемые сайты для размещения отчётов о сбоях:

- [GitHub Gist](https://gist.github.com/)
- [mclo.gs](https://mclo.gs/)
- [Pastebin](https://pastebin.com/)
