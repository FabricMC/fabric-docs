Базовые предметы могут зайти так далеко, что вы можете добавить предмет, который имеет интерактивность с миром при использовании.

Есть несколько ключевых классов которые вы должны понять, прежде чем будете рассматривать события с ванильными предметами.

## Класс InteractionResult {#interactionresult}

`InteractionResult` сообщает игре о статусе события: было ли оно пройдено/пропущено, завершилось ли оно неудачей или успехом.

Успешное взаимодействие (`InteractionResult.SUCCESS`) также можно использовать для изменения использованного предмета.

```java
ItemStack heldStack = user.getStackInHand(hand);
heldStack.decrement(1);
InteractionResult.SUCCESS.heldItemTransformedTo().success(heldStack);
```

## Переопределяемые события {#overridable-events}

К счастью, класс предмета имеет множество методов, которые можно переопределить, чтобы добавить дополнительную функциональность вашим предметам.

::: info

Отличный пример есть в статье [Воспроизведение SoundEvents](../sounds/using-sounds). Там метод `useOn` переопределяется, чтобы проигрывать звук при клике ПКМ по блоку.

:::

| Метод                  | Информация                                                                            |
| ---------------------- | ------------------------------------------------------------------------------------- |
| `hurtEnemy`            | Вызывается когда игрок, ударяет сущность.                             |
| `mineBlock`            | Вызывается когда игрок, ломает блок.                                  |
| `inventoryTick`        | Вызывается каждый тик когда предмет находится в инвентаре.            |
| `onCraftedPostProcess` | Вызывается когда предмет, создаётся.                                  |
| `useOn`                | Вызывается когда игрок кликает правой кнопкой мыши предметом на блок. |
| `use`                  | Вызывается когда игрок кликает правой кнопкой мыши предметом.         |
