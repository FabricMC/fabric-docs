Fabric – это легкий инструментарий модификации для Minecraft: Java Edition, разработанный, чтобы быть простым и легким в использовании. Это позволяет разработчикам применять модификации («моды») к стандартной игре, добавлять новые функции или изменять существующие механики.

Эта документация проведет вас через модификацию с помощью Fabric, от [создание вашего первого мода](./getting-started/creating-a-project) и [настройка вашей среды](./getting-started/setting-up), к сложным темам, как [сеть](./networking), [рендер](./rendering/basic-concepts), [сеть данных](./data-generation/setup) и т.д.

Прежде чем начать модификацию с помощью Fabric, вам нужно иметь определенное понимание разработки с помощью Java и объектно-ориентированного программирования в целом.

## Что предлагает Fabric? {#what-does-fabric-offer}

Проект Fabric сосредоточен вокруг трех основных компонентов:

- **Загрузчик Fabric**: гибкий, независимый от платформы загрузчик модов, в первую очередь разработанный для Minecraft: Java Edition
- **Fabric API**: дополнительный набор API и инструментов, которые разработчики моды могут использовать при создании модов
- **Fabric Loom**: плагин [Gradle](https://gradle.org/), позволяющий разработчикам легко разрабатывать и отлаживать моды

### Что предлагает Fabric API? {#what-does-fabric-api-offer}

Fabric API предоставляет широкий набор API, которые дополняют стандартные функции, позволяя как расширенную, так и упрощенную разработку.

Например, он предоставляет новые хуки, события, утилиты, такие как расширители доступа к функциям, доступ к встроенным реестрам, таким как реестр компостируемых предметов и многое другое.
