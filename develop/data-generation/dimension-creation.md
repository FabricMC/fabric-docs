The creation of Dimensions are broken down into 2 parts:

- **DimensionType**: This defines the features of the dimension. i.e. hasFixedTime, hasSkyLight, minY, etc.
- **Level Stem (Dimension)**: This defines the biomes, noise settings and dimensionType.

::: info INFO

A dimension may or may not have a custom biome. i.e. the user can choose to have singular or multiple vanilla biome.

:::

## Level Stem {#levelstem}

Unfortunately, Fabric API does not officially support the generation of files with `minecraft:dimension` registry that is used by the level stem. Although there are workarounds but those are out of the scope of this guide therefore, it needs to be created manually.

But we are fortunate enough to have amazing tools like [Dimension Generator](https://misode.github.io/dimension/) and the [Minecraft wiki](https://minecraft.wiki/w/Tutorial:Adding_a_new_dimension#Dimension) to aid in the creation of the level stem.

::: info INFO

Here is an example of level stem

@[code lang=json](@/reference/latest/src/main/resources/data/example-mod/dimension/example_dimension.json)

The value of `"type"` in the JSON can be replaced with other dimension types like `minecraft:overworld` to have their features and skip making a Dimension Type and the same applies to `"biome_source"`.

:::

::: warning

The dimension JSON should not be located in the generated folder as it is overridden every time data is generated.
Instead, it should be located in `src/main/resources/data/example-mod/dimension`. Replace `example-mod` with your mod ID.

:::

## Creating a World {#createworld}

After opening the create world menu and clicking on create world this appears:

![Experimental Feature Warning](/assets/develop/data-generation/dimension-generation/experimental_warning.png)

The warning screen will ask you to confirm using experimental world settings. It means that new versions can remove this feature to add custom dimensions without warning, which they probably won't. So you should click "Yes".

::: info Info

Another warning can appear when loading an existing world with or without the custom dimension:

![Experimental Features not Supported Warning](/assets/develop/data-generation/dimension-generation/expermental_settings_not_supported_warning.png)

You can press "I know what I'm doing!".

:::

### Teleporting to the Dimension {#dimensiontp}

Currently, we don't have any portal block or item or event setup to teleport us to the dimension. Therefore, to teleport to the dimension we will have to use this command:

`/execute in <mod-id>:<dimension> run tp <username> ~ ~ ~`

::: info INFO

the command in this guide will be:
`/execute in example-mod:example_dimension run tp Player986 ~ ~ ~`
