Leitfaden zur Fabric API 26.1 Portierung ​

Da Minecraft 26.1 nun nicht verschleichert ist, werden die meisten Entwickler nun die offiziellen Mappings von Mojang verwenden. Bisher wurde die Fabric-API unter Berücksichtigung der Yarn-Mappings entwickelt, aber mit der Umstellung auf offizielle Mappings wurden die API-Namen aktualisiert, um sie gegebenenfalls an die offiziellen Namen anzupassen. Diese Änderungen sind nicht abwärtskompatibel, daher musst du deinen Mod aktualisieren, um die neuen Namen verwenden zu können. Diese API-Umbenennungen werden nicht von den Tools zur Migration von Mappings verarbeitet. Es wird dringend empfohlen, zunächst die Migration von Mappings durchzuführen, wenn dein Mod mit Yarn-Mappings erstellt wurde.

Nachfolgend findest du eine Liste aller umbenannten Klassen, Methoden und Felder in der Fabric API 26.1. Am besten verwendest du diese Liste, indem du auf dieser Seite nach dem alten Namen suchst (⌘/STRG+F) und ihn durch den neuen Namen ersetzt.

Eine IntelliJ IDEA-Migrationskarte ist ebenfalls verfügbar, um die Umbenennung der Klassen zu automatisieren. Nach dem Herunterladen, folge den Anweisungen in der JetBrains-Dokumentation, um es auf dein Projekt anzuwenden.
